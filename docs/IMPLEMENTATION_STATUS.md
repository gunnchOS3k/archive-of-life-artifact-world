# Implementation Status

Generated by `npm run audit:implementation`. See `public/data/status/implementation_status.json`.

## Current snapshot scope (evidence-based)

After the **Real Data Completion Pass**, local systems are classified from audits and import reports:

| Status | Typical systems |
|--------|-----------------|
| **FULLY_IMPLEMENTED** | Game loop, museum, regions, ArchiveDex, Time Atlas, coverage matrix, dashboards, Python/SQL pipeline, source snapshot workflow, provenance model |
| **PARTIAL_IMPLEMENTATION** | NASA Earth Layer Console (until real metadata cached via `source:import:nasa`) |
| **BLOCKED_BY_EXTERNAL_DATA** | COL, GBIF, IUCN, PBDB, Neotoma full ingestion until snapshots imported |

**Note:** `FULLY_IMPLEMENTED` for ArchiveDex, Time Atlas, and coverage tools means **fully implemented for the current verified snapshot scope** — not global scientific completeness.

## Provenance classes

- `game_authored_verified` — original game content
- `source_verified` — imported external snapshot
- `mock_sample` — illustrative pipeline data
- `derived_inferred` — educated inference
- `blocked_external` — awaiting import
